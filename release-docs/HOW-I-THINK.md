# How I Think — evidence from Minewalker

| Strength | Evidence in this project |
|----------|---------------------------|
| Systems Thinker | Cover → Desk → play shell; single game-state owner with snapshot fan-out to HUD and scene; endless expand as a closed feedback loop |
| Product-Oriented Engineering | Training separated from levels; endless scored by depth not win; dig-by-facing as a deliberate presence tradeoff; deep-links optimized for handoff, not hub browsing |
| Clean Code Discipline | Mine rules kept engine-free; presentation mirrors state; decisions logged after recovery so intent survives the next session |

**Highlight:** The interesting problem wasn’t “render a grid in 3D” — it was keeping classic minesweeper honesty while making space, facing, and camera part of the puzzle.
